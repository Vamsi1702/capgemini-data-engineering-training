# Phase 3 — PySpark ETL Pipeline for Sales Analysis

## Project Overview

In this phase I built an ETL pipeline using PySpark to process raw sales and customer data, clean it, and generate business insights through transformations and aggregations. The code was written and tested on Spark Playground.

---

## Datasets Used

| Dataset | Description |
|---------|-------------|
| customers.csv | Customer details like customer_id, name, city |
| sales.csv | Transaction details like sale_id, customer_id, total_amount, sale_date |

---

## Technologies Used

- **PySpark** — for data processing and transformations
- **Spark SQL** — for SQL based querying on DataFrames
- **Spark Playground** — browser based environment to run PySpark code
- **GitHub** — for version control and storing the project

---

## ETL Pipeline Structure

**Extract** — Loaded customers.csv and sales.csv into PySpark DataFrames and explored the data using show() and printSchema()

**Transform** — Cleaned null values, removed invalid rows, fixed data types, and applied joins and aggregations to answer business questions

**Load** — Final results were displayed and output screenshots were saved in the outputs folder

---

## Exercises Completed

| Exercise | Business Question |
|----------|------------------|
| Exercise 1 | Total revenue generated each day |
| Exercise 2 | Total revenue generated by each city |
| Exercise 3 | Customers who placed more than 2 orders |
| Exercise 4 | Highest spending customer in each city |
| Exercise 5 | Final report with customer name, city, total spent and order count |

---

## Difficulties Faced

- Understanding JOIN syntax in PySpark — solved by practicing step by step
- Confusion between WHERE and HAVING — learned that HAVING is used after grouping
- Column naming errors causing AnalysisException — fixed by checking printSchema() carefully
- Data type issues — total_amount was loaded as string and needed to be cast to double

---

## What I Learned

- How to build an ETL pipeline from scratch using PySpark
- How to clean data by removing nulls and fixing data types before any calculation
- How to use JOIN operations to combine data from two different tables
- How to apply aggregate functions like SUM and COUNT using groupBy and agg()
- The difference between WHERE and HAVING and when to use each one
- How to convert SQL queries into PySpark code
- Importance of writing clean, commented and professional code
